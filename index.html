#Networking

Interactive reference guide

🌐
Networking = connecting devices so they can share data.
✅
Two must-remember words:
  • Node: Any device (phone, laptop, printer)
  • Communication link: The path data uses (Wi‑Fi or cable)

Devices in a network

🧠
Quick trick: End devices use data • Intermediary devices move data.

End devices (Use Data)

💻 Desktop / Laptop
📱 Mobile Phone
📲 Tablet
🖨️ Printer
🗄️ Server

Intermediary devices (Move Data)

These devices sit in the middle and help data reach the correct place.

🔄 Switch (Inside LAN)
Connects many devices in the same local network and forwards data to the correct device.
🔀 Router (Between networks)
Connects LAN ↔ internet and forwards data to the next network.
📶 Wireless Router
Router + Wi‑Fi in one device.
🔌 Modem
Connects your router to your ISP line (fiber/cable/DSL).
🗼 Cell Tower
Connects your phone to the mobile network (4G/5G).

Network Characteristics

These are the qualities of a good network.

1️⃣ Fault Tolerance
Meaning: The network keeps working even if one part fails.
Example: If one road is blocked, you take another road and still reach home.
2️⃣ Scalability
Meaning: The network can handle more users/devices later and still works well.
Example: Today 10 users, tomorrow 100 users — it should still be fast.
3️⃣ Quality of Service (QoS)
Meaning: The network can give priority to important traffic so it has less delay.
Example: Giving higher priority to a VoIP call over email sending so the voice remains clear.
4️⃣ Security
Meaning: Prevents unauthorized access and misuse. Governed by the CIA Triad:
  • Confidentiality: Only allowed people can see the data.
  • Integrity: Data is not changed wrongly.
  • Availability: Data/service is available when needed.

Data Flow

🚦
Data flow = direction of communication between 2 devices.
➡️ Simplex
One-way only.
Ex: Keyboard → CPU
↔️ Half‑duplex
Two-way, but one at a time.
Ex: Walkie‑talkie
🔁 Full‑duplex
Two-way, at the same time.
Ex: Phone call

Protocols

📜
Protocol = the rules that control communication. It decides What, How, and When data is communicated.

Elements of Protocols

📝 Message Encoding
How data is changed into signals (e.g., text/voice/video converted to bits).
📦 Message Format & Encapsulation
How data is packed. Data is wrapped with extra info (like “from”, “to”, “type”).
📏 Message Size
Breaking big data into smaller parts (chunks/packets).
⏱️ Message Timing
Controls the speed, waiting period, and order of delivery.
🎯 Message Delivery Options
  • Unicast: 1 sender → 1 receiver
  • Multicast: 1 sender → selected group
  • Broadcast: 1 sender → everyone in the network
