Networking Architecture

An interactive, animated reference guide to data communication.

Networking = connecting devices so they can share data.
Two must-remember words:
  • Node: Any device (phone, laptop, printer)
  • Communication link: The path data uses (Wi‑Fi or cable)

Intermediary Devices (Move Data)

These devices sit in the middle and route traffic to the correct destination.

Switch (Inside LAN)
Connects many devices in the same local network and forwards data to the correct device based on MAC address.
Router (Between networks)
Connects LAN ↔ Internet and forwards data to the next network based on IP address.
Wireless Router
Acts as a router, switch, and wireless access point in one device.
Modem
Modulates and demodulates signals; connects your router to your ISP line (fiber/cable/DSL).

Network Characteristics

The core qualities of a reliable and scalable network infrastructure.

Fault Tolerance

The network keeps working even if one part fails. If a link goes down, traffic is rerouted.

Scalability

Can handle more users or devices being added without experiencing a noticeable drop in performance.

Quality of Service (QoS)

Prioritizes time-sensitive traffic (like VoIP or video calls) over less urgent traffic (like emails).

Security (CIA Triad)

Confidentiality (encryption), Integrity (hashing), and Availability (uptime).
